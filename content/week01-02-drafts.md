## 【篇2】周日 3/23 — 公司号

**类型：** 作品集DIY-101

**标题：**
```
数媒作品集第一个project，怎么选题才不踩坑
```
备选：
```
零基础申数媒，第一个project选什么最保险
```

**正文：**
```
每年帮学生改作品集，最常见的问题不是技术不够——是选题太大、太空、太「正确」。

「探索人与科技的关系」
「关注气候变化」
「重新思考城市空间」

这些题目本身没错，但太多人在做同样的事，评审看麻了。

数媒/交互方向，第一个project选题的三个原则：

① 小而具体 > 大而正确
不要「探索记忆」——要「我奶奶的手机相册里有2000张食物照片，这说明了什么」

② 有个人经历锚点
评审记不住第100个「人与科技」，但能记住「在纽约唐人街长大的第二代移民如何用声音重建故乡」

③ 技术跟着概念走
先定「我要说什么」，再问「用什么工具能说出来」。反过来选技术，大概率做出来的是demo不是作品

第一个project不需要完美——需要有一个真实的出发点。

评论区说说你现在有什么想做的方向，我帮你看看。
```

当时申请时为了搞清楚这三个学校到底有什么区别，查了很久，写出来备忘。

NYU IDM：技术驱动，偏装置和系统。毕业生多在科技公司或做独立studio。作品集要展示你能用技术解决一个具体问题。

RCA（IDE/IED）：研究性最强，学术氛围重。适合想做PhD或者对设计理论有兴趣的人。

我自己的判断：如果你目标是毕业后做studio或接项目，IDM比RCA实际得多。如果你想留英工作，UAL的校友网络在伦敦更强。
